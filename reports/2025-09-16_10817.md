## 문제 요약

세 정수 중 두 번째로 큰 수(가운데 값)를 출력합니다. 입력된 세 수를 리스트로 모아 오름차순 정렬한 뒤, 가운데 인덱스(1)의 값을 출력합니다.

## 접근

- 세 수 A, B, C를 입력받아 리스트로 구성합니다.
- 리스트를 오름차순 정렬하면 [작은수, 가운데수, 큰수] 순이 됩니다.
- 가운데 인덱스 1의 값을 출력하면 두 번째로 큰 수를 얻습니다.
- 동률(같은 값)이나 음수도 정렬로 자연스럽게 처리됩니다.

## 시간·공간 복잡도

- 시간 복잡도: n=3에 대해 사실상 O(1). 일반적인 정렬의 이론적 복잡도는 O(n log n)이지만 n이 고정(3)이라 상수 시간입니다.
- 공간 복잡도: O(1) 추가 공간. 길이 3의 리스트만 사용합니다.

## 대안 접근

1) 수식 활용: 가운데 값 = (A + B + C) - max(A, B, C) - min(A, B, C). 정렬 없이 O(1) 시간/공간.
2) 조건문 비교: 세 수를 직접 비교하여 가운데 값을 도출(분기 다소 복잡하지만 O(1)).
3) heapq.nlargest(2, [A,B,C])[1]: 두 번째로 큰 값을 힙으로 구함(이 문제에선 과하지만 개념적으로 가능).

## 주석 강화 코드

```python
A, B, C = map(int, input().split())  # 한 줄에서 세 정수를 입력받음

# 세 수를 리스트에 담고 오름차순 정렬합니다.
# 정렬 결과: [작은수, 가운데수, 큰수]
# - 동률(같은 값)이 있어도 가운데 값은 올바르게 선택됩니다.
# - 음수도 비교 연산으로 올바르게 정렬됩니다.
# 복잡도 참고: 일반적으로 정렬은 O(n log n)이지만, 여기서는 n=3으로 사실상 O(1)입니다.
nums = [A, B, C]
nums.sort()

# 인덱스 1은 가운데 값 -> 세 수 중 두 번째로 큰 수
print(nums[1])
```
